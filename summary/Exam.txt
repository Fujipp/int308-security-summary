1) ภาพรวม Security
==================


เป้าหมายหลักของความมั่นคงปลอดภัย: CIA
-------------------------------------

- Confidentiality: ป้องกันข้อมูลรั่วไหลหรือถูกเปิดเผยแก่ผู้ไม่เกี่ยวข้อง
- Integrity: ป้องกันข้อมูลถูกแก้ไข ทำลาย หรือปลอมแปลงโดยไม่ถูกต้อง
- Availability: ระบบและข้อมูลต้องพร้อมใช้งานเมื่อจำเป็น

จำง่าย:
- Confidentiality = คนไม่ควรเห็น ต้องไม่เห็น
- Integrity = ข้อมูลต้องไม่เพี้ยน
- Availability = ระบบต้องไม่ล่มง่าย

ภัยคุกคามหลัก
-------------

- การหยุดชะงักของบริการ
- การทำลายข้อมูล
- ภัยพิบัติธรรมชาติ/มนุษย์
- การบุกรุกจากภายนอก
- Insider threat หรือพนักงานภายใน
- Virus / Worm / Malware
- DoS / DDoS

Risk Assessment
---------------

แนวคิดสำคัญคือ ประเมินความเสี่ยงก่อนเลือกมาตรการป้องกัน
- ระบุ asset ที่สำคัญ
- ระบุ threat
- ดูโอกาสเกิด + ผลกระทบ
- เทียบกับ control ที่มีอยู่
- ปรับมาตรการให้เหมาะกับความเสี่ยง

ข้อสอบชอบถาม:
- ระบบธนาคารออนไลน์กับเว็บร้านอาหาร ความเสี่ยงไม่เท่ากัน
- Security control ที่ดีต้องสัมพันธ์กับระดับความเสี่ยง

Security Policy
---------------

เป็นกฎ/แนวปฏิบัติที่กำหนดว่า
- อะไรคือทรัพย์สินสำคัญ
- ใครมีสิทธิ์อะไร
- ต้องป้องกันอย่างไร
- พนักงานต้องทำอะไร
- ต้องมีการฝึกอบรมและทดสอบเป็นระยะ

2) Cryptography พื้นฐาน
=======================


Cryptography คืออะไร
--------------------

การใช้หลักคณิตศาสตร์เพื่อป้องกันข้อมูลระหว่างส่งหรือขณะเก็บ

ให้บริการได้มากกว่าแค่ความลับ:
- Confidentiality
- Authentication
- Integrity
- บางกรณีรองรับ Non-repudiation

คำศัพท์สำคัญ
------------

- Plaintext: ข้อความต้นฉบับ
- Ciphertext: ข้อความที่เข้ารหัสแล้ว
- Cipher: อัลกอริทึมที่ใช้แปลงข้อมูล
- Key: ค่าลับที่ใช้ร่วมกับ cipher
- Encryption: แปลง plaintext → ciphertext
- Decryption: แปลง ciphertext → plaintext
- Cryptography: การออกแบบระบบเข้ารหัส
- Cryptanalysis: การพยายามถอด/โจมตีการเข้ารหัส
- Cryptology: รวมทั้ง cryptography + cryptanalysis

Ciphers vs Codes
----------------

- Cipher: ทำงานระดับตัวอักษร/บิต ด้วยอัลกอริทึม
- Code: แทนคำหรือวลีหนึ่งด้วยอีกคำหนึ่ง
ปัจจุบันใช้งานจริงเน้น cipher มากกว่า


========================================================================


3) Symmetric Encryption
=======================


แนวคิด
------

ใช้ กุญแจเดียวกัน ในการเข้ารหัสและถอดรหัส

ข้อดี:
- เร็ว
- เหมาะกับการเข้ารหัสข้อมูลจำนวนมาก

ข้อเสีย:
- ปัญหาใหญ่คือ การแจกจ่ายกุญแจ (key distribution)
- ถ้ากุญแจรั่ว ทั้งระบบจบ

ข้อกำหนดของ symmetric encryption ที่ปลอดภัย
-------------------------------------------

1. อัลกอริทึมต้องแข็งแรง
2. กุญแจต้องเป็นความลับ

วิธีโจมตี
---------

1. Cryptanalytic attack
~~~~~~~~~~~~~~~~~~~~~~~

ใช้ความรู้เกี่ยวกับโครงสร้างของข้อความ/อัลกอริทึมมาช่วยถอดรหัส

2. Brute force attack
~~~~~~~~~~~~~~~~~~~~~

ลองทุกคีย์ไปเรื่อย ๆ
- ยิ่งคีย์ยาว ยิ่งใช้เวลานาน
- นี่คือเหตุผลว่าทำไม key length สำคัญมาก

4) Classical Encryption
=======================


4.1 Caesar Cipher
-----------------

แทนแต่ละตัวอักษรด้วยตัวที่เลื่อนไป k ตำแหน่ง
- ง่ายมาก
- มีได้แค่ 26 แบบ
- แตกได้ด้วย brute force ง่ายมาก

4.2 Monoalphabetic Cipher
-------------------------

แทนแต่ละตัวอักษรด้วยตัวอื่นแบบคงที่
- มี key space ใหญ่มาก (26!)
- แต่ยังแตกได้ด้วย frequency analysis

4.3 Frequency Analysis
----------------------

แนวคิดสำคัญ:
- ภาษาอังกฤษมีตัวอักษรบางตัวออกบ่อย เช่น E, T, A
- ถ้าการเข้ารหัสยังคงความถี่เดิมไว้ ผู้โจมตีใช้สถิติช่วยถอดได้

4.4 Polyalphabetic Cipher / Vigenère Cipher
-------------------------------------------

ใช้หลายชุด alphabet ตาม keyword
- ยากกว่า monoalphabetic
- แต่ยังวิเคราะห์ได้ถ้าพบรูปแบบ

4.5 One-Time Pad
----------------

ใช้ key แบบสุ่มจริง และยาวเท่าข้อความ และใช้ครั้งเดียว
- ปลอดภัยเชิงทฤษฎีที่สุด
- แต่ใช้งานจริงลำบากมาก เพราะสร้างและแจกจ่าย key ยาก

4.6 Transposition Cipher
------------------------

ไม่ได้เปลี่ยนตัวอักษร แต่สลับตำแหน่ง
- ความถี่ตัวอักษรยังเหมือนเดิม
- จึงยังมีร่องรอยให้วิเคราะห์ได้

4.7 Product Cipher
------------------

เอา substitution + transposition มาผสมกัน
- เป็นรากฐานแนวคิดของ block cipher สมัยใหม่

5) Hashing
==========


Hash คืออะไร
------------

ฟังก์ชันที่รับข้อมูลยาวเท่าไรก็ได้ แล้วให้ผลลัพธ์ยาวคงที่

คุณสมบัติ:
- ผลลัพธ์สั้นและยาวคงที่
- ย้อนกลับไม่ได้
- เปลี่ยน input นิดเดียว hash เปลี่ยนมาก

Hashing vs Encryption
---------------------

- Encryption: ถอดกลับได้ถ้ามี key
- Hashing: ถอดกลับไม่ได้

การใช้งาน
---------

- ตรวจสอบ integrity
- เก็บ password แบบไม่เก็บ plaintext
- ใช้ร่วมกับ digital signature
- ใช้สร้าง MAC บางรูปแบบ

อัลกอริทึม hash
---------------

- MD5 (ไม่ปลอดภัยแล้ว)
- SHA-1 (ไม่ปลอดภัยแล้ว)
- SHA-224 / 256 / 384 / 512

คุณสมบัติที่ดีของ hash function
-------------------------------

- One-way
- Weak collision resistance
- Strong collision resistance

Birthday Attack
---------------

สาระสำคัญ:
- การหาค่า collision ง่ายกว่าที่คิด
- hash ยาว m บิต ความยากของ collision ประมาณ 2^(m/2)
- ดังนั้น hash สั้นเกินไปไม่ปลอดภัย

จำง่าย:
- hash 64 บิต ไม่ได้ปลอดภัยเท่าที่ดู
- ความยาก collision ไม่ใช่ 2^64 แต่ประมาณ 2^32


========================================================================


6) Block Cipher และแนวคิดสมัยใหม่
=================================


Block Cipher
------------

เข้ารหัสข้อมูลเป็นบล็อกขนาดคงที่ เช่น 64 หรือ 128 บิต

Stream Cipher
-------------

เข้ารหัสทีละบิต/ไบต์ เหมือนกระแสข้อมูล

Shannon: Confusion และ Diffusion
--------------------------------

สองแนวคิดสำคัญของ cipher ที่ดี
- Confusion: ทำให้ความสัมพันธ์ระหว่าง key กับ ciphertext ซับซ้อน
- Diffusion: กระจายผลของ plaintext ให้กระทบ ciphertext หลายส่วน

Feistel Structure
-----------------

ใช้ใน DES และ block cipher หลายตัว
- แบ่งข้อมูลเป็นซ้าย/ขวา
- ทำหลายรอบ
- แต่ละรอบใช้ round function + subkey
- ข้อดี: ถอดรหัสได้โดยใช้โครงสร้างเดียวกันกับการเข้ารหัส แค่สลับลำดับ subkey

องค์ประกอบที่มักถาม:
- block size
- key size
- number of rounds
- round function
- key schedule


========================================================================


7) DES
======


พื้นฐาน
-------

- Data Encryption Standard
- block size = 64 บิต
- key = 56 บิต
- ใช้ Feistel structure
- เคยเป็นมาตรฐานสำคัญมาก

โครงสร้างคร่าว ๆ
----------------

- Initial Permutation (IP)
- 16 rounds
- แต่ละรอบใช้ฟังก์ชัน F
- S-boxes 8 ตัว
- Final Permutation

จุดสำคัญของรอบใน DES
--------------------

ในแต่ละรอบ:
- ขยาย R จาก 32 → 48 บิต
- XOR กับ subkey
- ผ่าน S-box เพื่อลดกลับเป็น 32 บิต
- ผ่าน permutation P

จุดอ่อนของ DES
--------------

1. Key สั้นเกินไป
~~~~~~~~~~~~~~~~~

56 บิตปัจจุบัน brute force ได้

2. วิเคราะห์เชิงสถิติได้
~~~~~~~~~~~~~~~~~~~~~~~~

- Differential cryptanalysis
- Linear cryptanalysis

3. Timing attacks
~~~~~~~~~~~~~~~~~

โจมตี implementation ไม่ได้โจมตีแค่อัลกอริทึม

สรุปสอบ:
- DES สำคัญในเชิงประวัติศาสตร์
- แต่ ไม่เพียงพอสำหรับความปลอดภัยสมัยใหม่

8) Triple DES และ Contemporary Symmetric Ciphers
================================================


ทำไมต้อง Triple DES
-------------------

เพราะ DES เดี่ยวไม่ปลอดภัยแล้ว

Double DES ทำไมไม่พอ
--------------------

มี meet-in-the-middle attack
จึงลดความแข็งแรงลงมากกว่าที่คิด

Triple DES (3DES)
-----------------

ใช้การเข้ารหัส 3 รอบ เช่น EDE
- แบบ 2 keys
- แบบ 3 keys

ข้อดี:
- ปลอดภัยกว่า DES มาก

ข้อเสีย:
- ช้ากว่า AES
- block ยังเล็ก
- ปัจจุบัน AES เหนือกว่าในงานทั่วไป

9) Modes of Operation
=====================


Block cipher เข้ารหัสได้ทีละบล็อก แต่ข้อมูลจริงยาวไม่คงที่ จึงต้องมี mode

9.1 ECB (Electronic Codebook)
-----------------------------

- เข้ารหัสแต่ละ block แยกอิสระ
- block ที่ plaintext เหมือนกัน จะได้ ciphertext เหมือนกัน

ข้อเสีย:
- เห็น pattern ชัด
- ไม่เหมาะกับข้อมูลจำนวนมาก

ใช้เมื่อ:
- ข้อมูลสั้น ๆ หรือ block เดี่ยว

9.2 CBC (Cipher Block Chaining)
-------------------------------

- แต่ละ block จะ XOR กับ ciphertext block ก่อนหน้า
- ต้องมี IV

ข้อดี:
- ซ่อน pattern ดีกว่า ECB

ข้อเสีย:
- เข้ารหัสแบบขนานยาก
- ถ้า IV ถูกดัดแปลง อาจกระทบ block แรก

เหมาะกับ:
- bulk data encryption

9.3 CFB (Cipher Feedback)
-------------------------

- เปลี่ยน block cipher ให้คล้าย stream cipher
- ใช้ encryption mode ทั้งสองฝั่ง

ข้อดี:
- เหมาะกับข้อมูลที่มาเป็น stream

ข้อเสีย:
- error กระจายต่อหลายช่วง

9.4 OFB (Output Feedback)
-------------------------

- สร้าง keystream จากผล output ของ cipher
- ไม่ขึ้นกับ plaintext/ciphertext โดยตรง

ข้อดี:
- bit error ไม่กระจายต่อ

ข้อเสีย:
- ห้ามใช้ key+IV ซ้ำเด็ดขาด
- ต้อง sync กันตลอด

9.5 CTR (Counter Mode)
----------------------

- เข้ารหัสค่าตัวนับ แล้ว XOR กับ plaintext

ข้อดี:
- เร็ว
- ทำ parallel ได้
- random access ได้
- เหมาะกับงานความเร็วสูง

ข้อเสีย:
- ห้าม reuse key+counter เด็ดขาด

ออกสอบบ่อย:
- ECB แย่เพราะเห็น pattern
- CBC ต้องมี IV
- OFB/CTR ห้าม reuse keystream
- CTR เร็วและ parallel ได้ดี

10) Stream Cipher และ RC4
=========================


Stream Cipher
-------------

- เข้ารหัสทีละบิต/ไบต์
- XOR plaintext กับ keystream

ข้อควรจำที่สุด:
- ห้าม reuse keystream/key เดิม

RC4
---

- stream cipher ที่เคยนิยมมาก
- ใช้ permutation 0..255 เป็น state
- มี key scheduling และ pseudo-random generation

จุดสำคัญ:
- เร็วและเรียบง่าย
- แต่ปัจจุบัน ไม่แนะนำ ในระบบสมัยใหม่
- ปัญหาหลักมักอยู่ที่การจัดการ key/IV เช่นกรณี WEP

11) AES
=======


ทำไมต้องมี AES
--------------

DES ไม่ปลอดภัยแล้ว และ 3DES ช้า
NIST จึงคัดเลือก algorithm ใหม่ → Rijndael ได้เป็น AES

คุณสมบัติ AES
-------------

- symmetric block cipher
- block size = 128 บิต
- key = 128 / 192 / 256 บิต
- เร็วกว่า 3DES
- แข็งแรงกว่า DES

โครงสร้าง AES
-------------

AES ไม่ใช่ Feistel แต่เป็น iterative substitution-permutation network

ข้อมูลถูกเก็บเป็น state ขนาด 4x4 bytes

รอบหลัก ๆ:
1. SubBytes
2. ShiftRows
3. MixColumns
4. AddRoundKey

รอบสุดท้ายไม่มี MixColumns

อธิบายแต่ละขั้น
---------------

1. SubBytes
~~~~~~~~~~~

แทนค่าทุก byte ด้วย S-box
- เพิ่ม non-linearity
- ช่วยต้านการโจมตี

2. ShiftRows
~~~~~~~~~~~~

เลื่อนแต่ละแถวไม่เท่ากัน
- แถว 1 ไม่เลื่อน
- แถว 2 เลื่อน 1
- แถว 3 เลื่อน 2
- แถว 4 เลื่อน 3

3. MixColumns
~~~~~~~~~~~~~

ผสมค่าภายในแต่ละคอลัมน์ด้วยคณิตศาสตร์ใน GF(2^8)
- ช่วย diffusion

4. AddRoundKey
~~~~~~~~~~~~~~

XOR state กับ round key

Key Expansion
-------------

ขยาย key เดิมเป็นชุด round keys
จุดสำคัญ:
- ใช้ rotate
- ใช้ S-box
- ใช้ round constants

ทำไม AES แข็งแรง
----------------

- key ยาวกว่า DES มาก
- ออกแบบต้าน known attacks
- เร็วทั้ง software และ hardware
- เป็นมาตรฐานหลักในปัจจุบัน

สรุปสอบ:
- ถ้าถาม “gold standard ของ symmetric encryption ปัจจุบัน” → AES

12) Public-Key Cryptography
===========================


เหตุผลที่เกิดขึ้น
-----------------

Symmetric crypto มีปัญหาใหญ่เรื่อง key distribution
Public-key crypto ถูกพัฒนามาเพื่อแก้ปัญหานี้ และรองรับ digital signature

แนวคิด
------

ใช้ 2 กุญแจ:
- Public key: เปิดเผยได้
- Private key: ต้องเก็บเป็นความลับ

การใช้งาน:
- เข้ารหัสด้วย public key → ถอดด้วย private key
- เซ็นด้วย private key → ตรวจด้วย public key

การใช้งานหลัก 3 แบบ
-------------------

1. Encryption / Decryption
2. Digital Signature
3. Key Exchange

ข้อดี
-----

- ลดปัญหาการแลก secret key
- รองรับการพิสูจน์ตัวตนและลายเซ็นดิจิทัล

ข้อเสีย
-------

- ช้ากว่า symmetric crypto มาก
- ใช้กับข้อมูลจำนวนมากโดยตรงไม่คุ้ม

ดังนั้นของจริงมักใช้แบบ hybrid:
- ใช้ public key เพื่อแลก session key
- ใช้ symmetric key เข้ารหัสข้อมูลจริง

13) RSA
=======


หลักการ
-------

- คิดค้นโดย Rivest, Shamir, Adleman
- ความปลอดภัยอิงกับความยากของการแยกตัวประกอบจำนวนเต็มขนาดใหญ่

การสร้างกุญแจ
-------------

1. เลือก prime ใหญ่ 2 ตัว: p, q
2. คำนวณ n = p × q
3. คำนวณ φ(n) = (p−1)(q−1)
4. เลือก e ที่ gcd(e, φ(n)) = 1
5. หา d ที่ e·d ≡ 1 mod φ(n)
6. Public key = {e, n}
7. Private key = {d, n}

การเข้ารหัส/ถอดรหัส
-------------------

- Encryption: C = M^e mod n
- Decryption: M = C^d mod n

ทำไม RSA ใช้ได้
---------------

เพราะ Euler’s theorem และการเลือก e,d ให้เป็น modular inverse กัน

จุดเด่น
-------

- ใช้เข้ารหัส
- ใช้ทำ digital signature
- ใช้แลก session key

จุดอ่อน / การโจมตี
------------------

1. Brute force (โดยตรงแทบเป็นไปไม่ได้ถ้าคีย์ใหญ่พอ)
2. Mathematical attack / factoring
3. Timing attack
4. Chosen ciphertext attack

ข้อควรจำ
--------

- RSA ช้า
- ใช้กับข้อมูลก้อนเล็กหรือ session key
- ต้องใช้ padding ที่ปลอดภัย เช่น OAEP

14) Key Management
==================


ทำไมสำคัญ
---------

ต่อให้อัลกอริทึมดีแค่ไหน ถ้าจัดการ key ไม่ดี ระบบก็ไม่ปลอดภัย

การแจกจ่าย public key
---------------------

มีหลายแบบ:
1. Public announcement
   - ง่าย แต่ปลอมแปลงได้
2. Public directory
   - มีทะเบียนเก็บ key
   - แต่ต้องเชื่อถือ directory
3. Public-key authority
   - มี authority ช่วยยืนยัน key แบบ real-time
4. Public-key certificate
   - แนวทางสำคัญที่สุดในโลกจริง

Public-Key Certificate
----------------------

เอกสารที่ผูก
- ตัวตนของเจ้าของ
- public key
- validity period
- issuer
- และลายเซ็นของ CA

CA = Certificate Authority

Hybrid Key Distribution
-----------------------

โลกจริงนิยมใช้
- public key ช่วยแจก/แลก secret key
- จากนั้นใช้ symmetric encryption กับข้อมูลจริง

เพราะ:
- เร็วกว่า
- compatible กับระบบเดิม
- scale ได้ดี

15) Diffie-Hellman Key Exchange
===============================


จุดประสงค์
----------

ใช้แลก shared secret ผ่านช่องทางที่ไม่ปลอดภัย โดยไม่ต้องส่ง secret นั้นตรง ๆ

แนวคิด
------

- ตกลงค่า public ร่วมกัน เช่น q และ a
- แต่ละฝ่ายเลือก secret ของตัวเอง
- สร้าง public value ของตัวเอง
- แลก public values กัน
- คำนวณ shared key ได้เท่ากันทั้งสองฝั่ง

ความปลอดภัยอิงกับ:
- ความยากของ discrete logarithm problem

จุดสำคัญมาก
-----------

DH อย่างเดียวไม่ยืนยันตัวตน
จึงเสี่ยง man-in-the-middle attack

ดังนั้นต้องใช้ร่วมกับการพิสูจน์ตัวตน เช่น certificate หรือ signature

16) Message Authentication

สิ่งที่ต้องการ:

วิธีทำได้ 3 กลุ่ม

MAC (Message Authentication Code)

คุณสมบัติ:

- ไม่ใช่ digital signature

เพราะ MAC ใช้ secret key ร่วมกัน ทั้งผู้ส่งและผู้รับสร้างได้

คุณสมบัติที่ MAC ควรมี

- หา message อื่นที่ให้ MAC เดิมได้ยาก

- ขึ้นกับทุกบิตของข้อความ

17) Authentication Applications

17.1 Kerberos
-------------

ระบบ authentication แบบศูนย์กลางจาก MIT
ใช้ symmetric key และมี trusted third party

องค์ประกอบสำคัญ:
- AS = Authentication Server

Kerberos v5 ปรับปรุงจาก v4

- รองรับ algorithm มากขึ้น
- ปรับเรื่อง protocol/network/byte order
- ปรับปรุงเรื่อง lifetime, inter-realm, security

19) Email Security: PGP และ S/MIME
==================================


ปัญหาของ email ปกติ
-------------------

- ถูกดักอ่านได้
- ถูกแก้ไขได้
- ปลอมผู้ส่งได้

ต้องการ:
- confidentiality
- authentication
- integrity
- non-repudiation

PGP
---

ระบบ secure email ที่นิยมมาก

PGP authentication
~~~~~~~~~~~~~~~~~~

- hash ข้อความ
- ใช้ private key ของผู้ส่งเซ็น hash
- ผู้รับใช้ public key ตรวจสอบ

PGP confidentiality
~~~~~~~~~~~~~~~~~~~

- สร้าง session key แบบสุ่ม
- เข้ารหัสข้อความด้วย symmetric key
- เข้ารหัส session key ด้วย public key ของผู้รับ

=> เป็น hybrid cryptosystem ชัดเจน

จุดเสริมของ PGP
~~~~~~~~~~~~~~~

- บีบอัดข้อมูลก่อน encrypt
- แปลง binary เป็น ASCII ด้วย radix-64
- ตัดแบ่งข้อความถ้ายาวเกินไป

S/MIME
------

มาตรฐาน secure email บน MIME
รองรับ
- enveloped data
- signed data
- clear-signed data
- signed and enveloped data

ใช้ X.509 certificates

20) Database Security
=====================


พื้นฐาน
-------

Database คือทรัพยากรข้อมูลสำคัญมากขององค์กร
จึงต้องมีหลายชั้นป้องกัน เช่น
- firewall
- authentication
- access control
- DB access control
- encryption

SQL Injection (SQLi)
--------------------

หนึ่งในภัยที่สำคัญที่สุดของ web app

หลักการ:
- ผู้โจมตีส่ง SQL ที่เป็นอันตรายผ่าน input
- ทำให้ database รันคำสั่งที่ไม่ควรรัน

เป้าหมาย:
- ขโมยข้อมูล
- แก้ไข/ลบข้อมูล
- บางกรณีถึงขั้นรันคำสั่งระบบปฏิบัติการ
- ทำ DoS

ประเภทของ SQLi
--------------

1. Inband
~~~~~~~~~

ใช้ช่องทางเดียวกันทั้งโจมตีและรับผลลัพธ์

2. Inferential
~~~~~~~~~~~~~~

ไม่ได้เห็นข้อมูลตรง ๆ แต่เดาจากพฤติกรรมระบบ
- error-based
- blind SQL injection

3. Out-of-Band
~~~~~~~~~~~~~~

ดึงข้อมูลออกทางช่องทางอื่น

การป้องกัน SQLi
---------------

สไลด์เน้นว่าเป็นส่วนหนึ่งของ countermeasures โดยแนวคิดหลักคือ
- ควบคุม input อย่างเข้มงวด
- ออกแบบ query ให้ปลอดภัย
- จำกัดสิทธิ์ของ DB user
- แยก role ให้เหมาะสม

26) Reliability, Resiliency, Availability, Serviceability
=========================================================


หัวข้อนี้ออกสอบง่ายเพราะชอบถามความต่าง

Reliability
-----------

ความน่าจะเป็นที่ระบบจะทำงานได้โดยไม่ล้มเหลวในช่วงเวลาหนึ่ง
- เน้นว่า “พังบ่อยไหม”

Resiliency
----------

ความสามารถในการทนต่อ disruption, ปรับตัว, และฟื้นตัวได้เร็ว
- เน้น “โดนกระแทกแล้วอยู่รอดและกลับมาได้เร็วไหม”

Availability
------------

สัดส่วนเวลาที่ระบบพร้อมใช้งาน
- เน้น “พร้อมใช้มากแค่ไหน”

Serviceability
--------------

ความง่าย/เร็วในการซ่อมบำรุงหรือเปลี่ยนอุปกรณ์
- เน้น “พอพังแล้วซ่อมง่ายไหม”

จำแบบสั้น:
- Reliability = ไม่พัง
- Resiliency = พังแล้วรับมือได้
- Availability = ใช้งานได้ต่อเนื่อง
- Serviceability = ซ่อมง่าย

ค่าแนะนำจาก ASHRAE

PAC / Precision Air Conditioning

CRAC

- เหมาะกับ data center ใหญ่

31) Disaster Recovery (DR)
==========================


Disaster คืออะไร
----------------

การหยุดชะงักยาวนานของระบบ mission-critical จนกระทบธุรกิจอย่างมีนัยสำคัญ

บางธุรกิจไม่ต้องรอนาน:
- e-commerce
- โรงพยาบาล
- สายการบิน
แค่ไม่กี่ชั่วโมงก็ถือเป็น disaster ได้

HA vs DR
--------

High Availability (HA)
~~~~~~~~~~~~~~~~~~~~~~

- ป้องกันไม่ให้ระบบล่ม
- มักอยู่ใน data center เดียวกัน
- downtime ต้องต่ำมาก
- ใช้ clustering, load balancing

Disaster Recovery (DR)
~~~~~~~~~~~~~~~~~~~~~~

- เน้นกู้ระบบเมื่อเกิดเหตุใหญ่
- ข้าม location
- downtime ยังมีได้บ้างแต่ต้องอยู่ในเกณฑ์รับได้
- ใช้ replication, DR site

จำง่าย:
- HA = กันล้ม
- DR = ล้มแล้วกู้

การออกแบบ Disaster-Tolerant Architecture
----------------------------------------

- แยก location ระหว่าง primary DC กับ DR
- replicate ข้อมูล
- มี power redundancy
- มี network redundancy
- มี automatic failover
- มี backup/recovery system

ระยะห่างของ DR Site
-------------------

- ขั้นต่ำ 40–50 km
- best practice 120–160 km
- ถ้าไกลเกิน 300 km อาจมี latency สูง

ประเภท DR Site
--------------

- Cold site: มีแค่โครงสร้างพื้นฐาน
- Warm site: มีระบบบางส่วนและข้อมูลบางส่วน
- Hot site: พร้อมใช้งานเกือบเต็มรูปแบบ

RTO และ RPO
-----------

RTO (Recovery Time Objective)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

เวลาสูงสุดที่ยอมให้ระบบหยุดได้

RPO (Recovery Point Objective)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

ปริมาณข้อมูลสูญหายสูงสุดที่ยอมรับได้ วัดเป็นเวลา

จำง่าย:
- RTO = กู้ให้กลับมาภายในกี่ชั่วโมง
- RPO = ยอมเสียข้อมูลย้อนหลังได้กี่นาที/ชั่วโมง

Replication
-----------

Offline replication
~~~~~~~~~~~~~~~~~~~

- ใช้สื่อหรือ batch transfer
- ถูกกว่า
- recovery ช้ากว่า
- data อาจไม่ล่าสุด

Online replication
~~~~~~~~~~~~~~~~~~

#### Synchronous
- เขียนพร้อมกันทั้งสองฝั่ง
- data loss ต่ำมาก/เกือบศูนย์
- ต้องการ low latency

#### Asynchronous
- ส่งตามหลัง
- รองรับระยะไกลได้ดี
- อาจเสียข้อมูลเล็กน้อย

ระดับของ replication
--------------------

- Storage / block-level
- Database-level / log-based
- Application / file-level

32) เรื่องที่ควรเชื่อมโยงกันให้ได้
==================================


32.1 Encryption + Authentication + Integrity
--------------------------------------------

- Encryption อย่างเดียว ไม่ได้แปลว่า authentication ครบเสมอ
- Hash ใช้ตรวจ integrity
- MAC ใช้ authentication + integrity แต่ไม่ใช่ non-repudiation
- Digital signature ใช้ authentication + integrity + non-repudiation

32.2 Symmetric vs Public-Key
----------------------------

Symmetric
~~~~~~~~~

- เร็ว
- เหมาะกับข้อมูลเยอะ
- ติดเรื่อง key distribution

Public-key
~~~~~~~~~~

- แก้ปัญหา key distribution
- รองรับ signature
- ช้า

โลกจริง
~~~~~~~

ใช้ร่วมกันแบบ hybrid

32.3 DES vs 3DES vs AES
-----------------------

- DES: เก่า, key สั้น, ไม่พอ
- 3DES: ปลอดภัยขึ้น แต่ช้า
- AES: มาตรฐานปัจจุบัน เร็วและแข็งแรง

32.4 ECB vs CBC vs CTR
----------------------

- ECB: ง่ายแต่เห็น pattern
- CBC: ปลอดภัยกว่า ECB แต่ต้องมี IV
- CTR: เร็ว, parallel, ดีมากถ้าไม่ reuse counter

32.5 Kerberos vs X.509
----------------------

- Kerberos: symmetric, trusted central server, ticket-based
- X.509: public-key, certificate-based, ใช้ใน PKI

32.6 HA vs DR
-------------

- HA เน้นไม่ให้บริการสะดุด
- DR เน้นฟื้นตัวจากเหตุใหญ่

33) จุดที่มักออกสอบแบบถามตรง
============================


1. CIA คืออะไร และยกตัวอย่างแต่ละตัว
2. ความต่างระหว่าง cipher กับ code
3. ความต่างระหว่าง encryption กับ hashing
4. ทำไม MD5/SHA-1 ไม่ควรใช้
5. ทำไม One-Time Pad ปลอดภัยมากแต่ใช้งานจริงยาก
6. ความหมายของ confusion และ diffusion
7. ทำไม DES ไม่ปลอดภัยแล้ว
8. ทำไม Double DES ไม่พอ และต้องใช้ 3DES
9. เปรียบเทียบ ECB, CBC, CFB, OFB, CTR
10. ทำไม reuse key/IV/counter ใน OFB/CTR/stream cipher อันตราย
11. โครงสร้างหลักของ AES
12. ทำไมระบบจริงใช้ hybrid cryptosystem
13. หลักการของ RSA
14. Diffie-Hellman แลก key ได้อย่างไร และจุดอ่อนคืออะไร
15. MAC ต่างจาก digital signature อย่างไร
16. Kerberos ทำงานผ่าน AS/TGS/Ticket อย่างไร
17. X.509 certificate มีข้อมูลอะไรบ้าง
18. CRL คืออะไร
19. ประเภทของ SQL Injection
20. ทำไม RBAC ช่วยด้าน security และ administration
21. ความต่างระหว่าง Reliability / Resiliency / Availability / Serviceability
22. ความต่างของ HAC vs CAC
23. ความต่างของ HA vs DR
24. RTO และ RPO ต่างกันอย่างไร
25. ความต่างระหว่าง Cold / Warm / Hot site

34) สรุปสุดท้ายแบบจำให้ขึ้นหัว
==============================


- ความมั่นคงปลอดภัยพื้นฐาน = CIA
- Symmetric เร็ว แต่ติด key distribution
- Public-key ช่วยเรื่อง key distribution และ signature แต่ ช้า
- โลกจริงใช้ hybrid encryption
- DES เก่าแล้ว, 3DES เป็นทางผ่าน, AES คือมาตรฐานหลัก
- Hash ใช้ตรวจความถูกต้อง, MAC ใช้ยืนยันข้อความแบบใช้ key ร่วม, Digital signature ใช้ยืนยันตัวตนพร้อม non-repudiation
- Diffie-Hellman ใช้แลก key แต่ต้องมี authentication เพิ่ม ไม่งั้นโดน MITM ได้
- Kerberos = ticket-based, symmetric, central server
- X.509 = certificate-based, public key, PKI
- Database security ต้องกัน SQLi, จำกัดสิทธิ์, ใช้ RBAC, และ encryption
- Data center security ไม่ใช่แค่ cyber แต่รวม power, cooling, fire, physical security, monitoring
- HA เน้นไม่ล่ม, DR เน้นกู้กลับมา, วัดด้วย RTO/RPO

35) สูตรจำสั้น ๆ ก่อนเข้าห้องสอบ
================================


- CIA = Confidentiality, Integrity, Availability
- RSA = asymmetric, based on factoring
- DH = key exchange, vulnerable to MITM if unauthenticated
- MAC ≠ Digital Signature
- ECB bad for patterns
- CTR fast + parallel, never reuse counter
- AES > 3DES > DES ในการใช้งานสมัยใหม่
- RTO = เวลากู้, RPO = ข้อมูลที่ยอมเสียได้
- Reliability = ไม่พัง, Availability = พร้อมใช้, Resiliency = ฟื้นตัวเก่ง, Serviceability = ซ่อมง่าย
