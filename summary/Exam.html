1) INT308 Introduction

  • Security เป้าหมายหลักคือ CIA: Confidentiality, Integrity, Availability
  • เริ่มคิดจาก Risk ก่อนเลือกมาตรการป้องกัน
  • Security policy คือกฎว่าใครทำอะไรได้ และต้องปฏิบัติอย่างไร
  • Threat หลักที่ควรจำ: malware, insider threat, DoS/DDoS, data theft, disaster
C = กันข้อมูลรั่ว
I = กันข้อมูลถูกแก้
A = ระบบพร้อมใช้

1) INT308 Introduction

  • The core security goals are the CIA triad: Confidentiality, Integrity, Availability
  • Start with risk before choosing controls
  • A security policy defines who can do what and how they must behave
  • Key threats to remember: malware, insider threat, DoS/DDoS, data theft, and disasters

2) Cryptography

  • การเข้ารหัสช่วยปกป้องข้อมูลระหว่างส่งและระหว่างเก็บ
  • คำสำคัญ: plaintext, ciphertext, key, encryption, decryption
  • รู้ความต่างระหว่าง cipher กับ code
  • ด้านที่ออกสอบบ่อย: symmetric, brute force, cryptanalysis
  • ต้องจำให้ได้ว่า encryption ให้ confidentiality ส่วน hash ให้ integrity

ประเด็นจำง่าย: ถ้าอยากเก็บความลับ ใช้ encryption. ถ้าอยากตรวจความถูกต้อง ใช้ hash หรือ MAC.

จำแบบสั้น

Encryption = ซ่อนข้อมูล

Hash = ตรวจว่าข้อมูลเปลี่ยนไหม

MAC = ตรวจข้อมูล + เช็กคนส่ง

2) Cryptography

  • Cryptography protects data in transit and at rest
  • Key terms: plaintext, ciphertext, key, encryption, decryption
  • Know the difference between a cipher and a code
  • Common exam topics: symmetric encryption, brute force, and cryptanalysis
  • Remember: encryption gives confidentiality, while hash gives integrity

3) Block Ciphers

  • Caesar, monoalphabetic, Vigenère, transposition คือพื้นฐานที่ต้องรู้
  • One-time pad ปลอดภัยเชิงทฤษฎี แต่ใช้งานจริงยาก
  • Product cipher คือการผสม substitution + transposition
  • Confusion และ diffusion คือแนวคิดสำคัญของ cipher ที่ดี
  • โจทย์ชอบถาม frequency analysis, key space, และข้อจำกัดของ classical ciphers

3) Block Ciphers

  • Know the basics: Caesar, monoalphabetic, Vigenère, and transposition
  • One-time pad is theoretically perfect but impractical
  • A product cipher combines substitution and transposition
  • Confusion and diffusion are key design goals for strong ciphers
  • Expect questions about frequency analysis, key space, and limits of classical ciphers

4) AES

  • AES เป็นมาตรฐาน symmetric ที่ใช้จริงในปัจจุบัน
  • ทำงานกับ block ขนาด 128 บิต และใช้ key 128/192/256 บิต
  • รอบหลักประกอบด้วย SubBytes, ShiftRows, MixColumns, AddRoundKey
  • จำให้ได้ว่า AES เร็ว ปลอดภัย และใช้แพร่หลายมาก

ถ้าข้อสอบถามว่า symmetric cipher ที่ควรตอบในปัจจุบันคืออะไร ให้ตอบ AES เป็นหลัก

โครงสร้าง AES แบบจำเร็ว

SubBytesShiftRowsMixColumnsAddRoundKey

ให้คิดว่า AES คือการสลับ + กระจาย + ผสม + ใส่กุญแจในแต่ละรอบ

4) AES

  • AES is the standard symmetric cipher used today
  • It uses 128-bit blocks and 128/192/256-bit keys
  • Main round steps: SubBytes, ShiftRows, MixColumns, AddRoundKey
  • Remember: AES is fast, secure, and widely deployed

5) CSC / Modes of Operation

  • โหมดที่ชอบออก: ECB, CBC, CFB, OFB, CTR
  • ECB ไม่แนะนำ เพราะข้อมูลรูปแบบเดิมยังเห็นได้
  • CBC ต้องมี IV และใช้กันแพร่หลาย
  • CTR เร็ว และขนานได้ดี
  • จำให้ได้ว่า OFB/CTR สร้าง keystream และห้ามใช้ซ้ำ

5) CSC / Modes of Operation

  • Common modes: ECB, CBC, CFB, OFB, CTR
  • ECB is not recommended because patterns remain visible
  • CBC requires an IV and is widely used
  • CTR is fast and highly parallelizable
  • Remember: OFB/CTR generate a keystream and must never be reused

6) RSA

  • เป็น public-key cryptography ใช้คู่กุญแจ public/private
  • เด่นเรื่อง key exchange และ digital signature
  • ช้ากว่า symmetric จึงมักใช้แบบ hybrid
  • จำหลักการ: ความปลอดภัยมาจากความยากของการ factor จำนวนมาก
RSA ใช้ทำอะไร

ใช้ส่งกุญแจหรือเซ็นชื่อดิจิทัล มากกว่าการเข้ารหัสข้อมูลจำนวนมาก

6) RSA

  • RSA is a public-key cryptosystem using public/private keys
  • It is useful for key exchange and digital signatures
  • It is slower than symmetric crypto, so it is often used in hybrid systems
  • Its security relies on the difficulty of factoring large numbers

7) Key Management

  • ระบบเข้ารหัสจะปลอดภัยจริงเมื่อ จัดการกุญแจดี
  • PKI และ X.509 certificate ใช้ผูก identity กับ public key
  • CA คือผู้รับรอง ส่วน CRL/OCSP ใช้ตรวจการเพิกถอน
  • หัวข้อนี้มักถามเรื่อง trust chain และ certificate validity
เส้นทางความเชื่อถือ

ผู้ใช้เชื่อ certificate เพราะ CA รับรอง แล้ว browser/system ตรวจ chain และ revocation อีกชั้น

7) Key Management

  • Encryption is only as strong as the key management behind it
  • PKI and X.509 certificates bind identities to public keys
  • CA issues trust, while CRL/OCSP handle revocation checks
  • Expect questions about trust chains and certificate validity

8) Authentication

  • Kerberos ใช้ ticket-based authentication เหมาะกับองค์กร
  • ลดการส่งรหัสผ่านซ้ำ ๆ บนเครือข่าย
  • การยืนยันตัวตนที่ดีต้องแยกให้ได้ระหว่าง authentication, authorization, และ accounting
  • จำให้ได้ว่า system time สำคัญมากใน Kerberos

Kerberos = มี trusted server กลาง + ticket. X.509 = certificate + public key + PKI.

Kerberos สรุป 3 คำ

Ticket, timestamp, trusted server

8) Authentication

  • Kerberos uses ticket-based authentication and fits enterprise networks
  • It reduces repeated password transmission
  • Know the difference between authentication, authorization, and accounting
  • System time matters a lot in Kerberos

9) Data Center Security

  • Data center ต้องคิดเรื่องไฟฟ้า เครือข่าย ระบบระบายความร้อน และการเข้าถึง
  • หัวข้อสำคัญ: availability, resiliency, reliability, serviceability
  • มีแนวคิดอย่าง raised floor, NOC, hot aisle / cold aisle
  • จำว่า cooling และ redundancy สำคัญต่อ uptime
จำความต่าง 4 คำ

Reliability = ไม่พังบ่อย

Availability = พร้อมใช้

Resiliency = ฟื้นตัวได้ดี

Serviceability = ซ่อมง่าย

9) Data Center Security

  • Data centers must consider power, network, cooling, and physical access
  • Key ideas: availability, resiliency, reliability, and serviceability
  • Topics include raised floors, NOC, and hot/cold aisle design
  • Cooling and redundancy are critical for uptime

10) Database & Cloud Security

  • SQL Injection คือภัยหลักของ database security
  • ป้องกันด้วย prepared statements, parameterized queries, least privilege, RBAC
  • Cloud ต้องเข้าใจ shared responsibility และ data protection
  • DR กับ HA ต่างกัน: HA ลด downtime, DR เน้นกู้คืนเมื่อเกิดเหตุใหญ่
  • DB encryption เป็น last line of defense แต่ไม่แทน access control

ถ้าจำได้แค่ 3 คำ: parameterized query, least privilege, shared responsibility

SQLi ป้องกันยังไง

อย่าต่อ string query เอง ใช้ parameterized query ทุกครั้ง และให้สิทธิ์ฐานข้อมูลเท่าที่จำเป็น

10) Database & Cloud Security

  • SQL injection is the main database security threat
  • Use prepared statements, parameterized queries, least privilege, and RBAC
  • Cloud security depends on the shared responsibility model and data protection
  • HA and DR are different: HA minimizes downtime, DR focuses on recovery after major incidents
  • Database encryption is a last line of defense, not a replacement for access control
